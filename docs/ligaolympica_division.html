Cross-Member Dependencies

Who depends on whom — coordinate these interfaces carefully

Rovpoli's code is used by

  • KD→ All character classes extend GameCharacter
  • Biji→ All character classes extend GameCharacter
  • Selwyn→ Menu.java creates Battle objects
  • M5→ GameWindow creates BattlePanel which runs the battle

Theme.java (M5) is used by

  • Rovpoli→ BattlePanel.java imports Theme.*
  • KD→ PixelArt.java references Theme colors
  • Biji→ AnimatedBar.java uses Theme.darker()
  • Selwyn→ CharacterSelectPanel, RosterPanel import Theme.*

PixelArt.java (KD) is used by

  • Rovpoli→ BattlePanel.ArenaCanvas calls getSprite()
  • Biji→ PixelArt must have sprites for Biji's characters
  • Selwyn→ CharacterSelectPanel and RosterPanel call getSprite()

CharacterSelectPanel (Selwyn) used by

  • Rovpoli→ BattlePanel calls makeButton() factory from CharacterSelectPanel
  • M5→ GameWindow instantiates CharacterSelectPanel for both P1 and P2 flows

AnimatedBar (Biji) used by

  • Rovpoli→ BattlePanel.ArenaCanvas creates and reads AnimatedBar via reflection
  • Selwyn→ CharacterSelectPanel and RosterPanel instantiate AnimatedBar for stat display

Critical shared interfaces

  • setStunned(boolean) — BattlePanel calls this; all character classes must expose it via GameCharacter
  • resetForNewRound() — Called by both Battle.java and BattlePanel.java
  • autoTakeTurn(target) — Called by BattlePanel for AI mode; each character inherits from GameCharacter
  • skill1/2/3Cooldown fields — BattlePanel reads via reflection; must remain named exactly as-is in GameCharacter

Git Workflow Recommendation

  • One branch per member — e.g. feat/rovpoli-battle, feat/kd-sprites
  • Merge to main after Phase 1 — GameCharacter and Theme must land first
  • Pull before starting work each day — GameCharacter changes affect everyone
  • Never rename GameCharacter fields — BattlePanel uses reflection on cooldown field names
  • Run a full compile after every merge — catch interface mismatches early

Compile Order

  • 1st — ligaolympica.character (interface + base class)
  • 2nd — ligaolympica.character.* (concrete classes)
  • 3rd — ligaolympica.battle (depends on characters)
  • 4th — ligaolympica.ui (depends on battle + characters)
  • 5th — ligaolympica.gui (Theme first, then all others)
  • 6th — ligaolympica root (Main, Game)
