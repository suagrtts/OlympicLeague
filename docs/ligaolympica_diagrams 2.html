Project Overview

A mythology-themed turn-based arena fighter built in Java with rich OOP design

⚔️
Combat Engine
Best-of-3 round system with turn alternation, mana management, cooldown tracking, status effects (stun, evasion, buffs), and true damage that bypasses mitigation.
🏛️
Mythology Theme
Characters draw power from Greek and Norse gods — Zeus, Poseidon, Artemis, Ares, Apollo, Hermes, and Loki. Each God Gift ability is thematically distinct.
🎮
Three hello.Game Modes
PvP (two human players share keyboard), PvE (fight AI with random skill selection), and Arcade Mode (gauntlet with 3 difficulty tiers, health restores between fights).
🏗️
OOP Architecture
CharacterInterface → GameCharacter (base) → 9 concrete hero classes. Clean package separation: character, battle, ui, and root. Polymorphism used throughout.
🤖
AI System
autoTakeTurn() provides simple but functional computer opponents — selects random skills and falls back intelligently when skills are on cooldown.
🎯
Skill Variety
Characters feature unique mechanics: evasion, healing, shields, stun, attack buffs, damage immunity (Vanish), double strikes (Hermes' Speed), and true damage ignoring defenses.

Key Design Patterns

  • Template Method — GameCharacter defines the algorithm skeleton; subclasses override skill1/2/3 and takeTurn.
  • Strategy (light) — autoTakeTurn vs takeTurn swaps human/AI behavior polymorphically.
  • Polymorphism — Battle only knows GameCharacter; all concrete classes plug in seamlessly.
  • Interface Segregation — CharacterInterface declares the full contract; GameCharacter provides defaults; subclasses specialize.

Status Effect System

  • attackBonus — Multiplier applied in randomDamage(); resets after statusEffectTurns reach 0.
  • defenseBonus — Applied in takeDamage(); values below 1.0 = damage reduction.
  • untargetable — Completely ignores incoming damage for 1 turn (Vanish, Temporal Shift).
  • isStunned — Skips the character's next turn entirely; mana still restores.
  • nextAttackEvaded — Character-specific field (Atalyn, GoatedKit) that voids the next hit.

Primary Use Cases

  • Launch hello.Game — Splash screen displays, hello.Main calls hello.Game.start(), Menu is shown.
  • Browse Character Info — Player views any character's backstory, HP, MP, and all 3 skills.
  • Start PvP hello.Game — Both players pick characters; Battle.startBattle(false) runs.
  • Start PvE hello.Game — Player picks; AI randomly picks from 9 chars; startBattle(true).
  • Start Arcade Mode — Player picks character and difficulty; fights a shuffled gauntlet of opponents.
  • Select Character — selectCharacter() method presents numbered roster, returns new instance.

Battle & Extension Use Cases

  • Execute Turn-Based Round — Alternating attacker/defender, up to 40 turns; stun handling; mana restore each turn.
  • Choose & Use Skill — Player enters 1/2/3; system validates cooldown; executes and sets cooldown.
  • Manage Cooldowns/Effects — updateTurnEffects() decrements all cooldowns and expiring buffs each turn.
  • Escape Battle — «extend» on any turn; sets hasEscaped=true; startBattle returns false.
  • AI autoTakeTurn — «extend»; random skill pick with cooldown-aware fallback.
  • Receive Partial HP Restore — Arcade only; 75% max HP restored between each fight (except final).

System Architecture

Package structure, data flow, and design decisions

Package Structure

  • rootligaolympica — hello.Main.java (splash + entry), hello.Game.java (start wrapper)
  • characterligaolympica.character — CharacterInterface, GameCharacter, 9 hero classes
  • battleligaolympica.battle — Battle.java: all combat logic, round management, AI mode
  • uiligaolympica.ui — Menu.java: all user interaction, game mode routing

hello.Game Flow Summary

  • hello.Main → splash screen → hello.Game.start() → Menu.showMainMenu()
  • Menu gathers input → creates Battle(p1, p2) → calls startBattle()
  • Battle loops: reset, turn-swap, stun check, updateTurnEffects, takeTurn/autoTakeTurn
  • Round ends when HP drops to 0 or 40 turns pass; track wins1/wins2 first to 2
  • Arcade: arcadeBattle() loop → 75% HP restore after each win → final banner
  • Escape: hasEscaped flag set → startBattle returns false → Menu handles gracefully

Damage Formula

  • Step 1 — baseDamage ± variance (random within range)
  • Step 2 — × (0.85 + 0.3 × random) ≈ 85–115% natural variance
  • Step 3 — × attackBonus (buff multiplier, default 1.0)
  • Step 4 — if defenseBonus active: × defenseBonus (reduction)
  • True Damage — skips steps 3 & 4 entirely; always full damage
randomDamage() attackBonus defenseBonus takeTrueDamage()

Cooldown System

  • Each character has skill1Cooldown, skill2Cooldown, skill3Cooldown (int).
  • On skill use: set cooldownN = N (turns to wait).
  • Each turn: updateTurnEffects() decrements all non-zero cooldowns by 1.
  • takeTurn() checks cooldown before executing; shows error if active.
  • autoTakeTurn() gracefully falls back to an available skill if chosen is on CD.
  • resetForNewRound() zeros all cooldowns (best-of-3 resets fully each round).
skill1/2/3Cooldown updateTurnEffects() resetForNewRound()

🟢 EASY (4 opponents)
  • Vor, Orris, Orven, Achiron
  • Shuffled random order each run
  • Lower-tier damage output
